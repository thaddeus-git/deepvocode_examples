Change Date Format in ExtJS: Complete Tutorial with 3 Solutions

Quick Answer

To change date format from YYYY-m-d to d-m-YYYY in ExtJS, use Ext.Date.format():

var dt = new Date('2012-05-31');
var formattedDate = Ext.Date.format(dt, 'd-m-Y');
console.log(formattedDate); // Output: 31-05-2012

  • 🎨 Output Focused → Method 3 with templates (best presentation control)
  • ⚡ Legacy Maintainer → Method 1 with version compatibility (works across ExtJS versions)

Method 1: Ext.Date.format (Recommended)

Rating: ⭐ Recommended

How it works:

The Ext.Date.format() method is ExtJS’s primary date formatting utility. It accepts a Date object and a format string, returning a formatted string representation. This method handles timezone considerations and provides consistent formatting across different browsers.

The format string uses specific characters:

  • d = Day of month with leading zeros (01-31)
  • m = Month with leading zeros (01-12)
  • Y = Full year (e.g., 2012)
  • y = Two-digit year (e.g., 12)

Complete Implementation:

// Basic formatting function
function formatDateFromDatabase(dateValue) {
    // Handle different input types
    var dateObj;
    
    if (Ext.isString(dateValue)) {
        // Parse string date from database
        dateObj = new Date(dateValue);
    } else if (Ext.isDate(dateValue)) {
        // Already a Date object
        dateObj = dateValue;
    } else {
        // Invalid input
        return '';
    }
    
    // Validate the date
    if (isNaN(dateObj.getTime())) {
        console.warn('Invalid date provided:', dateValue);
        return '';
    }
    
    // Format to d-m-YYYY
    return Ext.Date.format(dateObj, 'd-m-Y');
}
 
// Enhanced version with error handling and caching
var DateFormatter = {
    cache: {},
    
    format: function(dateValue, fromFormat, toFormat) {
        fromFormat = fromFormat || 'Y-m-d';
        toFormat = toFormat || 'd-m-Y';
        
        // Create cache key
        var cacheKey = dateValue + '_' + fromFormat + '_' + toFormat;
        
        // Check cache first
        if (this.cache[cacheKey]) {
            return this.cache[cacheKey];
        }
        
        try {
            var dateObj;
            
            if (Ext.isString(dateValue)) {
                // Parse with specific format if needed
                if (fromFormat === 'Y-m-d') {
                    dateObj = new Date(dateValue);
                } else {
                    // Custom parsing for other formats
                    dateObj = Ext.Date.parse(dateValue, fromFormat);
                }
            } else {
                dateObj = dateValue;
            }
            
            if (!dateObj || isNaN(dateObj.getTime())) {
                return '';
            }
            
            var result = Ext.Date.format(dateObj, toFormat);
            
            // Cache the result
            this.cache[cacheKey] = result;
            
            return result;
            
        } catch (e) {
            console.error('Date formatting error:', e);
            return '';
        }
    },
    
    clearCache: function() {
        this.cache = {};
    }
};
 
// Usage examples
var dbDate1 = '2012-05-31';
var dbDate2 = '2023-12-25';
 
console.log(DateFormatter.format(dbDate1)); // 31-05-2012
console.log(DateFormatter.format(dbDate2)); // 25-12-2023
 
// Custom formats
console.log(DateFormatter.format(dbDate1, 'Y-m-d', 'M d, Y')); // May 31, 2012

Pros:

  • Works across all ExtJS versions (3.x to 7.x)
  • Handles timezone considerations automatically
  • Extensive format string options
  • Built-in error handling for invalid dates
  • Consistent cross-browser behavior

Cons:

  • Requires creating Date objects (memory overhead for large datasets)
  • No built-in caching mechanism
  • May need additional validation for edge cases

When to use:

  • Converting individual dates from database format
  • Building reusable formatting utilities
  • When you need maximum control over date formatting
  • Working with various date sources and formats

Method 2: Grid Column Renderer

Rating: ⭐ Highly Recommended for Grids

How it works:

Grid column renderers provide a clean way to format data for display without modifying the underlying data model. The renderer function receives the raw value and returns the formatted display value. This approach separates data storage from presentation, following ExtJS best practices.

Complete Implementation:

// Basic grid with date formatting
Ext.define('DateGrid', {
    extend: 'Ext.grid.Panel',
    
    initComponent: function() {
        // Store configuration
        this.store = Ext.create('Ext.data.Store', {
            fields: [
                'id',
                {name: 'dateField', type: 'date', dateFormat: 'Y-m-d'},
                'description'
            ],
            data: [
                {id: 1, dateField: '2012-05-31', description: 'First record'},
                {id: 2, dateField: '2023-12-25', description: 'Second record'},
                {id: 3, dateField: '2024-01-15', description: 'Third record'}
            ]
        });
        
        // Column configuration with custom renderer
        this.columns = [
            {text: 'ID', dataIndex: 'id', width: 50},
            {
                text: 'Date (d-m-Y)',
                dataIndex: 'dateField',
                width: 120,
                renderer: this.formatDateRenderer
            },
            {
                text: 'Date (Custom)',
                dataIndex: 'dateField',
                width: 150,
                renderer: function(value, metaData, record) {
                    return this.formatDateWithStyle(value, metaData);
                },
                scope: this
            },
            {text: 'Description', dataIndex: 'description', flex: 1}
        ];
        
        this.callParent(arguments);
    },
    
    // Basic date renderer
    formatDateRenderer: function(value) {
        if (!value) return '';
        
        var dateObj = Ext.isDate(value) ? value : new Date(value);
        
        if (isNaN(dateObj.getTime())) {
            return 'Invalid Date';
        }
        
        return Ext.Date.format(dateObj, 'd-m-Y');
    },
    
    // Advanced renderer with styling
    formatDateWithStyle: function(value, metaData) {
        if (!value) return '';
        
        var dateObj = Ext.isDate(value) ? value : new Date(value);
        
        if (isNaN(dateObj.getTime())) {
            metaData.style = 'color: red;';
            return 'Invalid Date';
        }
        
        var formattedDate = Ext.Date.format(dateObj, 'd-m-Y');
        var today = new Date();
        
        // Style based on date comparison
        if (dateObj > today) {
            metaData.style = 'color: green; font-weight: bold;';
            metaData.tdAttr = 'data-qtip="Future date"';
        } else if (Ext.Date.diff(today, dateObj, 'days') < 30) {
            metaData.style = 'color: orange;';
            metaData.tdAttr = 'data-qtip="Recent date"';
        }
        
        return formattedDate;
    }
});
 
// Reusable renderer factory
var DateRenderers = {
    // Create a renderer for specific format
    createRenderer: function(format, options) {
        options = options || {};
        
        return function(value, metaData, record) {
            if (!value) return options.emptyText || '';
            
            var dateObj = Ext.isDate(value) ? value : new Date(value);
            
            if (isNaN(dateObj.getTime())) {
                if (options.showInvalid) {
                    metaData.style = 'color: red;';
                    return 'Invalid Date';
                }
                return '';
            }
            
            var formatted = Ext.Date.format(dateObj, format);
            
            // Apply custom styling
            if (options.styleFunction) {
                options.styleFunction(dateObj, metaData, formatted);
            }
            
            return formatted;
        };
    },
    
    // Pre-built common renderers
    ddmmyyyy: function(value) {
        return DateRenderers.createRenderer('d-m-Y')(value);
    },
    
    mmddyyyy: function(value) {
        return DateRenderers.createRenderer('m-d-Y')(value);
    },
    
    longDate: function(value) {
        return DateRenderers.createRenderer('l, F j, Y')(value);
    }
};
 
// Usage in grid columns
var gridConfig = {
    columns: [
        {
            text: 'Date',
            dataIndex: 'dateField',
            renderer: DateRenderers.createRenderer('d-m-Y', {
                emptyText: 'No date',
                showInvalid: true,
                styleFunction: function(dateObj, metaData, formatted) {
                    if (dateObj < new Date()) {
                        metaData.style = 'color: gray;';
                    }
                }
            })
        }
    ]
};

  • Requires understanding of ExtJS grid architecture
  • More complex setup than simple formatting

Method 3: Template Formatting

Rating: ⭐ Excellent for Templates

How it works:

ExtJS templates and XTemplates provide built-in date formatting through the date member function. This approach is ideal when generating HTML content dynamically or working with complex layouts that include formatted dates.

// Advanced template with custom date functions
var advancedTemplate = new Ext.XTemplate(
    '<div class="event-card">',
        '<div class="date-section">',
            '<span class="day">{dateField:this.getDay}</span>',
            '<span class="month">{dateField:this.getMonth}</span>',
            '<span class="year">{dateField:this.getYear}</span>',
        '</div>',
        '<div class="content">',
            '<h4>{title}</h4>',
            '<p>Scheduled: {dateField:this.formatCustomDate}</p>',
            '<p class="{dateField:this.getDateClass}">',
                'Status: {dateField:this.getDateStatus}',
            '</p>',
        '</div>',
    '</div>',
    {
        // Custom template functions
        getDay: function(dateValue) {
            if (!dateValue) return '';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            return Ext.Date.format(dt, 'd');
        },
        
        getMonth: function(dateValue) {
            if (!dateValue) return '';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            return Ext.Date.format(dt, 'M');
        },
        
        getYear: function(dateValue) {
            if (!dateValue) return '';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            return Ext.Date.format(dt, 'Y');
        },
        
        formatCustomDate: function(dateValue) {
            if (!dateValue) return 'Not scheduled';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            return Ext.Date.format(dt, 'd-m-Y \\a\\t H:i');
        },
        
        getDateClass: function(dateValue) {
            if (!dateValue) return 'no-date';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            var now = new Date();
            
            if (dt < now) return 'past-date';
            if (Ext.Date.diff(now, dt, 'days') <= 7) return 'upcoming-date';
            return 'future-date';
        },
        
        getDateStatus: function(dateValue) {
            if (!dateValue) return 'No date set';
            var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
            var now = new Date();
            
            if (dt < now) return 'Past event';
            if (Ext.Date.diff(now, dt, 'days') <= 7) return 'Coming soon';
            return 'Future event';
        }
    }
);

                '</div>',

            {
                getRelativeDate: function(dateValue) {
                    if (!dateValue) return 'Unknown';
                    var dt = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
                    var now = new Date();
                    var diffDays = Ext.Date.diff(dt, now, 'days');
                    
                    if (diffDays === 0) return 'Today';
                    if (diffDays === 1) return 'Yesterday';
                    if (diffDays === -1) return 'Tomorrow';
                    if (diffDays > 0) return diffDays + ' days ago';
                    return Math.abs(diffDays) + ' days from now';
                }
            }
        );
        
        this.callParent(arguments);
    }
});
 
// Utility for creating date templates
var DateTemplateFactory = {
    create: function(format, options) {
        options = Ext.apply({
            wrapperClass: 'date-wrapper',
            emptyText: 'No date'
        }, options);
        
        var templateStr = '<span class="' + options.wrapperClass + '">';
        
        if (options.emptyText) {
            templateStr += '<tpl if="' + options.field + '">';
        }
        
        templateStr += '{' + options.field + ':date("' + format + '")}';
        
        if (options.emptyText) {
            templateStr += '<tpl else>' + options.emptyText + '</tpl>';
        }
        
        templateStr += '</span>';
        
        return new Ext.XTemplate(templateStr);
    },
    
    // Pre-built templates
    shortDate: function(field) {
        return this.create('d-m-Y', {field: field});
    },
    
    longDate: function(field) {
        return this.create('l, F j, Y', {field: field});
    },
    
    timeOnly: function(field) {
        return this.create('H:i', {field: field});
    }
};

Common Problems & Solutions

Problem 1: Invalid Date Objects

Symptoms: Getting “Invalid Date” or NaN errors when formatting

Solution:

function safeFormatDate(dateValue, format) {
    format = format || 'd-m-Y';
    
    // Handle null/undefined
    if (!dateValue) {
        return '';
    }
    
    // Convert to Date object
    var dateObj;
    if (Ext.isString(dateValue)) {
        dateObj = new Date(dateValue);
    } else if (Ext.isDate(dateValue)) {
        dateObj = dateValue;
    } else {
        console.warn('Invalid date input:', dateValue);
        return '';
    }
    
    // Validate the date
    if (isNaN(dateObj.getTime())) {
        console.warn('Could not parse date:', dateValue);
        return '';
    }
    
    return Ext.Date.format(dateObj, format);
}

Problem 2: Timezone Issues

Symptoms: Dates showing differently than expected, off by hours

Solution:

function formatDateWithTimezone(dateValue, format, timezone) {
    if (!dateValue) return '';
    
    var dateObj = Ext.isDate(dateValue) ? dateValue : new Date(dateValue);
    
    if (timezone) {
        // Adjust for specific timezone
        var offset = dateObj.getTimezoneOffset();
        var targetOffset = this.getTimezoneOffset(timezone);
        var diff = targetOffset - offset;
        dateObj = new Date(dateObj.getTime() + (diff * 60000));
    }
    
    return Ext.Date.format(dateObj, format || 'd-m-Y');
}

Problem 3: Performance with Large Datasets

Symptoms: Slow grid rendering or UI freezing with many date columns

Solution:

// Cached renderer for better performance
var CachedDateRenderer = function() {
    var cache = {};
    
    return function(value, metaData, record) {
        if (!value) return '';
        
        var cacheKey = value.toString();
        if (cache[cacheKey]) {
            return cache[cacheKey];
        }
        
        var dateObj = Ext.isDate(value) ? value : new Date(value);
        var result = Ext.Date.format(dateObj, 'd-m-Y');
        
        // Cache with size limit
        if (Object.keys(cache).length < 1000) {
            cache[cacheKey] = result;
        }
        
        return result;
    };
}();

Problem 4: Inconsistent Date Parsing

Symptoms: Different browsers parsing dates differently

Solution:

function parseConsistentDate(dateString) {
    // Handle YYYY-MM-DD format consistently
    if (typeof dateString === 'string' && /^\d{4}-\d{1,2}-\d{1,2}$/.test(dateString)) {
        var parts = dateString.split('-');
        var year = parseInt(parts[0], 10);
        var month = parseInt(parts[1], 10) - 1; // Month is 0-indexed
        var day = parseInt(parts[2], 10);
        
        return new Date(year, month, day);
    }
    
    // Fallback to standard parsing
    return new Date(dateString);
}

// Optimized renderer for high-frequency updates
var OptimizedDateRenderer = (function() {
    var formatCache = {};
    var lastClearTime = Date.now();
    
    return function(value) {
        // Clear cache every 5 minutes
        if (Date.now() - lastClearTime > 300000) {
            formatCache = {};
            lastClearTime = Date.now();
        }
        
        if (!value) return '';
        
        var key = value.toString();
        if (formatCache[key]) {
            return formatCache[key];
        }
        
        var result = Ext.Date.format(new Date(value), 'd-m-Y');
        formatCache[key] = result;
        
        return result;
    };
})();

Frequently Asked Questions

Q: Which method should I choose for my project?

A: Choose based on your context:

  • Simple formatting: Use Ext.Date.format() directly
  • Grid display: Use column renderers for better performance
  • HTML generation: Use XTemplate with date formatting
  • Large datasets: Implement caching with any method
  • Legacy support: Stick with Ext.Date.format() for maximum compatibility

Q: What about browser compatibility?

A: All methods shown work across modern browsers. ExtJS handles cross-browser differences internally. For IE8 and below (if using ExtJS 3.x), ensure you’re using the compatibility layer and test thoroughly.

Q: How do I handle edge cases like invalid dates?

A: Always validate dates before formatting:

function safeFormat(dateValue) {
    if (!dateValue) return '';
    var date = new Date(dateValue);
    return isNaN(date.getTime()) ? '' : Ext.Date.format(date, 'd-m-Y');
}

Q: Are there performance differences between methods?

A: Yes:

  • Direct formatting: Fast for small datasets
  • Grid renderers: Optimized for large datasets
  • Templates: Good for complex layouts but slower for simple formatting
  • Caching: Significantly improves performance for repeated formatting

Q: Can I combine multiple approaches?

A: Absolutely! You can use different methods in different parts of your application:

// Use direct formatting for simple cases
var simpleDate = Ext.Date.format(new Date(), 'd-m-Y');
 
// Use renderers in grids
var gridColumn = {
    renderer: function(value) {
        return Ext.Date.format(new Date(value), 'd-m-Y');
    }
};
 
// Use templates for complex layouts
var template = new Ext.XTemplate('{date:date("d-m-Y")}');

Documentation References

ExtJS Date Formatting Characters:

  • d - Day of month, 2 digits with leading zeros (01-31)
  • j - Day of month without leading zeros (1-31)
  • m - Month, 2 digits with leading zeros (01-12)
  • n - Month without leading zeros (1-12)
  • Y - Full year, 4 digits (e.g., 2012)
  • y - Year, 2 digits (e.g., 12)

Common Format Patterns:

  • d-m-Y → 31-05-2012
  • m/d/Y → 05/31/2012
  • Y-m-d → 2012-05-31
  • l, F j, Y → Thursday, May 31, 2012

Grid Renderer Best Practices:

  • Always check for null/undefined values
  • Use Ext.isDate() to validate date objects
  • Consider caching for performance
  • Apply conditional styling through metaData parameter
