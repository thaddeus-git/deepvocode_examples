How to Set RTC as Wake Up Source and Set a Specific Time on All Days Using Sys File System: Complete Tutorial with 3 Solutions

Quick Answer

To set RTC as a wake-up source for a specific time daily on your iMX6 SabreSD, use the /sys/class/rtc/rtc0/wakealarm interface. For midnight wake-up every day:

# Clear existing alarm
echo 0 > /sys/class/rtc/rtc0/wakealarm
# Set wake-up for next midnight
date +%s -d 'tomorrow 00:00:00' > /sys/class/rtc/rtc0/wakealarm

Method 1: Direct Sys Interface (Recommended)

Rating: ⭐ Recommended

How it works:

The Linux RTC subsystem exposes wake-up functionality through the sysfs interface at /sys/class/rtc/rtc0/wakealarm. This method directly manipulates this interface to set absolute wake-up times. The RTC hardware maintains the alarm even when the system is powered down, making it perfect for scheduled wake-ups.

The process involves:

  1. Clearing any existing alarm by writing 0 to wakealarm
  2. Calculating the target wake-up time as a Unix timestamp
  3. Writing the timestamp to wakealarm
  4. The RTC hardware triggers a wake-up signal at the specified time

Complete Implementation:

#!/bin/bash
# rtc_wakeup_direct.sh - Direct RTC wake-up configuration
# Compatible with iMX6 SabreSD and most ARM platforms
 
# Function to set RTC wake-up for specific time
set_rtc_wakeup() {
    local target_time="$1"
    local description="$2"
    
    # Validate RTC device exists
    if [ ! -f /sys/class/rtc/rtc0/wakealarm ]; then
        echo "Error: RTC device not found at /sys/class/rtc/rtc0/"
        return 1
    fi
    
    # Clear existing alarm (required before setting new one)
    echo "Clearing existing RTC alarm..."
    echo 0 > /sys/class/rtc/rtc0/wakealarm
    
    # Calculate target timestamp
    local target_timestamp
    target_timestamp=$(date +%s -d "$target_time")
    
    if [ $? -ne 0 ]; then
        echo "Error: Invalid time format '$target_time'"
        return 1
    fi
    
    # Verify target is in the future
    local current_timestamp=$(date +%s)
    if [ $target_timestamp -le $current_timestamp ]; then
        echo "Error: Target time must be in the future"
        return 1
    fi
    
    # Set the alarm
    echo "Setting RTC alarm for: $description"
    echo $target_timestamp > /sys/class/rtc/rtc0/wakealarm
    
    # Verify alarm was set
    local set_alarm=$(cat /sys/class/rtc/rtc0/wakealarm)
    if [ "$set_alarm" = "$target_timestamp" ]; then
        echo "✓ RTC alarm successfully set"
        echo "  Target time: $(date -d @$target_timestamp)"
        echo "  Seconds from now: $((target_timestamp - current_timestamp))"
        return 0
    else
        echo "Error: Failed to set RTC alarm"
        return 1
    fi
}
 
# Main execution
case "${1:-midnight}" in
    "midnight")
        set_rtc_wakeup "tomorrow 00:00:00" "Next midnight"
        ;;
    "custom")
        if [ -z "$2" ]; then
            echo "Usage: $0 custom 'time_specification'"
            echo "Example: $0 custom 'tomorrow 06:30:00'"
            exit 1
        fi
        set_rtc_wakeup "$2" "Custom time: $2"
        ;;
    "test")
        # Set wake-up in 2 minutes for testing
        set_rtc_wakeup "$(date -d '+2 minutes' '+%Y-%m-%d %H:%M:%S')" "Test wake-up (2 minutes)"
        ;;
    *)
        echo "Usage: $0 [midnight|custom 'time'|test]"
        echo "Examples:"
        echo "  $0 midnight                    # Wake at next midnight"
        echo "  $0 custom 'tomorrow 06:30:00' # Wake at 6:30 AM tomorrow"
        echo "  $0 test                        # Test wake-up in 2 minutes"
        exit 1
        ;;
esac

Pros:

  • Immediate execution with minimal overhead
  • Direct hardware interface, maximum reliability
  • Works with all Linux kernel versions supporting RTC
  • No additional dependencies or services required
  • Perfect for embedded systems like iMX6 SabreSD

Cons:

  • Requires manual re-execution for recurring alarms
  • No built-in error recovery or monitoring
  • Limited to single alarm at a time
  • Requires root privileges for sysfs access

When to use:

  • One-time wake-up events
  • Testing RTC functionality
  • Simple embedded applications
  • When you need guaranteed minimal resource usage
  • Development and debugging scenarios

Method 2: Script-based Solution

Rating: ⭐⭐⭐ Best for Automation

How it works:

This method combines the direct sysfs interface with intelligent scripting to provide automated, recurring wake-up functionality. It uses cron for scheduling and includes comprehensive error handling, logging, and status monitoring. The script automatically calculates the next occurrence of the target time and handles edge cases like daylight saving time transitions.

Complete Implementation:

#!/bin/bash
# rtc_wakeup_manager.sh - Advanced RTC wake-up management
# Supports recurring alarms, logging, and error recovery
 
# Configuration
SCRIPT_NAME="rtc_wakeup_manager"
LOG_FILE="/var/log/${SCRIPT_NAME}.log"
PID_FILE="/var/run/${SCRIPT_NAME}.pid"
CONFIG_FILE="/etc/${SCRIPT_NAME}.conf"
RTC_DEVICE="/sys/class/rtc/rtc0/wakealarm"
 
# Default configuration
DEFAULT_WAKE_TIME="00:00:00"
DEFAULT_LOG_LEVEL="INFO"
DEFAULT_MAX_LOG_SIZE="10M"
 
# Logging function
log_message() {
    local level="$1"
    local message="$2"
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[$timestamp] [$level] $message" | tee -a "$LOG_FILE"
}
 
# Load configuration
load_config() {
    if [ -f "$CONFIG_FILE" ]; then
        source "$CONFIG_FILE"
        log_message "INFO" "Configuration loaded from $CONFIG_FILE"
    else
        # Create default configuration
        cat > "$CONFIG_FILE" << EOF
# RTC Wake-up Manager Configuration
WAKE_TIME="$DEFAULT_WAKE_TIME"
LOG_LEVEL="$DEFAULT_LOG_LEVEL"
MAX_LOG_SIZE="$DEFAULT_MAX_LOG_SIZE"
ENABLE_RECURRING="true"
RETRY_ATTEMPTS="3"
RETRY_DELAY="5"
EOF
        log_message "INFO" "Default configuration created at $CONFIG_FILE"
    fi
    
    # Set defaults for missing values
    WAKE_TIME="${WAKE_TIME:-$DEFAULT_WAKE_TIME}"
    LOG_LEVEL="${LOG_LEVEL:-$DEFAULT_LOG_LEVEL}"
    MAX_LOG_SIZE="${MAX_LOG_SIZE:-$DEFAULT_MAX_LOG_SIZE}"
    ENABLE_RECURRING="${ENABLE_RECURRING:-true}"
    RETRY_ATTEMPTS="${RETRY_ATTEMPTS:-3}"
    RETRY_DELAY="${RETRY_DELAY:-5}"
}
 
# Validate RTC device and permissions
validate_environment() {
    # Check if running as root
    if [ "$EUID" -ne 0 ]; then
        log_message "ERROR" "Script must be run as root for RTC access"
        return 1
    fi
    
    # Check RTC device
    if [ ! -f "$RTC_DEVICE" ]; then
        log_message "ERROR" "RTC device not found: $RTC_DEVICE"
        return 1
    fi
    
    # Test RTC write access
    if ! echo 0 > "$RTC_DEVICE" 2>/dev/null; then
        log_message "ERROR" "Cannot write to RTC device: $RTC_DEVICE"
        return 1
    fi
    
    log_message "INFO" "Environment validation passed"
    return 0
}
 
# Calculate next occurrence of target time
calculate_next_wakeup() {
    local target_time="$1"
    local current_timestamp=$(date +%s)
    local today_target=$(date +%s -d "today $target_time" 2>/dev/null)
    
    if [ $? -ne 0 ]; then
        log_message "ERROR" "Invalid time format: $target_time"
        return 1
    fi
    
    # If today's target time has passed, use tomorrow
    if [ $today_target -le $current_timestamp ]; then
        local tomorrow_target=$(date +%s -d "tomorrow $target_time")
        echo $tomorrow_target
        log_message "DEBUG" "Today's target passed, using tomorrow: $(date -d @$tomorrow_target)"
    else
        echo $today_target
        log_message "DEBUG" "Using today's target: $(date -d @$today_target)"
    fi
    
    return 0
}
 
# Set RTC alarm with retry logic
set_rtc_alarm() {
    local target_timestamp="$1"
    local attempt=1
    
    while [ $attempt -le $RETRY_ATTEMPTS ]; do
        log_message "INFO" "Setting RTC alarm (attempt $attempt/$RETRY_ATTEMPTS)"
        
        # Clear existing alarm
        if echo 0 > "$RTC_DEVICE" 2>/dev/null; then
            # Set new alarm
            if echo $target_timestamp > "$RTC_DEVICE" 2>/dev/null; then
                # Verify alarm was set correctly
                local set_alarm=$(cat "$RTC_DEVICE" 2>/dev/null)
                if [ "$set_alarm" = "$target_timestamp" ]; then
                    log_message "INFO" "RTC alarm set successfully for: $(date -d @$target_timestamp)"
                    return 0
                else
                    log_message "WARN" "Alarm verification failed: expected $target_timestamp, got $set_alarm"
                fi
            else
                log_message "WARN" "Failed to write alarm timestamp"
            fi
        else
            log_message "WARN" "Failed to clear existing alarm"
        fi
        
        attempt=$((attempt + 1))
        if [ $attempt -le $RETRY_ATTEMPTS ]; then
            log_message "INFO" "Retrying in $RETRY_DELAY seconds..."
            sleep $RETRY_DELAY
        fi
    done
    
    log_message "ERROR" "Failed to set RTC alarm after $RETRY_ATTEMPTS attempts"
    return 1
}
 
# Schedule next wake-up via cron
schedule_recurring() {
    local wake_time="$1"
    
    # Create cron entry to run this script daily
    local cron_entry="0 0 * * * $0 --set-alarm '$wake_time' >/dev/null 2>&1"
    
    # Check if entry already exists
    if crontab -l 2>/dev/null | grep -q "$SCRIPT_NAME"; then
        log_message "INFO" "Cron entry already exists for recurring wake-up"
    else
        # Add cron entry
        (crontab -l 2>/dev/null; echo "# $SCRIPT_NAME recurring wake-up"; echo "$cron_entry") | crontab -
        log_message "INFO" "Added cron entry for recurring wake-up at $wake_time"
    fi
}
 
# Main alarm setting function
set_alarm() {
    local target_time="$1"
    
    log_message "INFO" "Starting RTC alarm setup for time: $target_time"
    
    # Validate environment
    if ! validate_environment; then
        return 1
    fi
    
    # Calculate next wake-up time
    local next_wakeup
    next_wakeup=$(calculate_next_wakeup "$target_time")
    if [ $? -ne 0 ]; then
        return 1
    fi
    
    # Set the alarm
    if set_rtc_alarm "$next_wakeup"; then
        log_message "INFO" "RTC wake-up configured successfully"
        
        # Schedule recurring if enabled
        if [ "$ENABLE_RECURRING" = "true" ]; then
            schedule_recurring "$target_time"
        fi
        
        return 0
    else
        log_message "ERROR" "Failed to configure RTC wake-up"
        return 1
    fi
}
 
# Status reporting
show_status() {
    echo "=== RTC Wake-up Manager Status ==="
    echo "RTC Device: $RTC_DEVICE"
    echo "Configuration: $CONFIG_FILE"
    echo "Log File: $LOG_FILE"
    echo ""
    
    if [ -f "$RTC_DEVICE" ]; then
        local current_alarm=$(cat "$RTC_DEVICE" 2>/dev/null)
        if [ -n "$current_alarm" ] && [ "$current_alarm" != "0" ]; then
            echo "Current Alarm: $(date -d @$current_alarm 2>/dev/null || echo 'Invalid timestamp')"
            echo "Time Until Alarm: $((current_alarm - $(date +%s))) seconds"
        else
            echo "Current Alarm: None set"
        fi
    else
        echo "RTC Device: Not accessible"
    fi
    
    echo ""
    echo "Recent Log Entries:"
    if [ -f "$LOG_FILE" ]; then
        tail -5 "$LOG_FILE"
    else
        echo "No log file found"
    fi
}
 
# Main script logic
main() {
    # Load configuration
    load_config
    
    case "${1:-help}" in
        "--set-alarm")
            set_alarm "${2:-$WAKE_TIME}"
            ;;
        "--status")
            show_status
            ;;
        "--clear")
            echo 0 > "$RTC_DEVICE" 2>/dev/null
            log_message "INFO" "RTC alarm cleared"
            ;;
        "--test")
            # Set alarm for 2 minutes from now
            local test_time=$(date -d '+2 minutes' '+%H:%M:%S')
            set_alarm "$test_time"
            ;;
        *)
            echo "Usage: $0 [--set-alarm 'HH:MM:SS'] [--status] [--clear] [--test]"
            echo ""
            echo "Examples:"
            echo "  $0 --set-alarm '00:00:00'  # Set wake-up for midnight"
            echo "  $0 --set-alarm '06:30:00'  # Set wake-up for 6:30 AM"
            echo "  $0 --status                # Show current status"
            echo "  $0 --clear                 # Clear current alarm"
            echo "  $0 --test                  # Test with 2-minute alarm"
            echo ""
            echo "Configuration file: $CONFIG_FILE"
            ;;
    esac
}
 
# Execute main function
main "$@"

Pros:

  • Automated recurring wake-ups without manual intervention
  • Comprehensive error handling and retry logic
  • Detailed logging for troubleshooting and monitoring
  • Configurable through external configuration file
  • Cron integration for hands-off operation
  • Status reporting and alarm verification

Cons:

  • More complex setup and configuration
  • Requires cron service to be running
  • Additional disk space for logs and configuration
  • May be overkill for simple one-time alarms

When to use:

  • Production systems requiring reliable daily wake-ups
  • Unattended embedded devices
  • Systems requiring detailed logging and monitoring
  • Applications needing automatic error recovery
  • When you need recurring alarms without manual intervention

Method 3: Service-based Approach

Rating: ⭐⭐⭐⭐ Enterprise Grade

How it works:

This method implements RTC wake-up management as a proper Linux systemd service, providing enterprise-grade reliability, monitoring, and integration with system management tools. The service runs continuously, monitoring the RTC state and automatically rescheduling wake-ups. It includes health checks, automatic recovery, and integration with system logging.

Complete Implementation:

#!/bin/bash
# rtc-wakeup-service.sh - Systemd service for RTC wake-up management
# Enterprise-grade RTC wake-up with monitoring and auto-recovery
 
SERVICE_NAME="rtc-wakeup"
SERVICE_USER="rtc-service"
WORK_DIR="/opt/rtc-wakeup"
CONFIG_DIR="/etc/rtc-wakeup"
LOG_DIR="/var/log/rtc-wakeup"
SYSTEMD_DIR="/etc/systemd/system"
 
# Service configuration
create_service_config() {
    cat > "$CONFIG_DIR/service.conf" << 'EOF'
# RTC Wake-up Service Configuration
[General]
WakeTime=00:00:00
CheckInterval=300
MaxRetries=5
RetryDelay=10
EnableHealthCheck=true
HealthCheckInterval=60
 
[Logging]
LogLevel=INFO
MaxLogFiles=10
MaxLogSize=10M
SyslogFacility=daemon
 
[Monitoring]
EnableMetrics=true
MetricsPort=9090
AlertThreshold=3
EOF
}
 
# Main service script
create_service_script() {
    cat > "$WORK_DIR/rtc-wakeup-daemon.py" << 'EOF'
#!/usr/bin/env python3
"""
RTC Wake-up Service Daemon
Enterprise-grade RTC wake-up management with monitoring
"""
 
import os
import sys
import time
import signal
import logging
import configparser
import subprocess
import threading
from datetime import datetime, timedelta
from pathlib import Path
 
class RTCWakeupService:
    def __init__(self, config_file='/etc/rtc-wakeup/service.conf'):
        self.config_file = config_file
        self.running = False
        self.config = configparser.ConfigParser()
        self.rtc_device = '/sys/class/rtc/rtc0/wakealarm'
        self.health_status = {'last_check': None, 'status': 'unknown', 'errors': 0}
        
        # Load configuration
        self.load_config()
        
        # Setup logging
        self.setup_logging()
        
        # Setup signal handlers
        signal.signal(signal.SIGTERM, self.signal_handler)
        signal.signal(signal.SIGINT, self.signal_handler)
        
        self.logger.info("RTC Wake-up Service initialized")
    
    def load_config(self):
        """Load service configuration"""
        try:
            self.config.read(self.config_file)
            
            # General settings
            self.wake_time = self.config.get('General', 'WakeTime', fallback='00:00:00')
            self.check_interval = self.config.getint('General', 'CheckInterval', fallback=300)
            self.max_retries = self.config.getint('General', 'MaxRetries', fallback=5)
            self.retry_delay = self.config.getint('General', 'RetryDelay', fallback=10)
            self.enable_health_check = self.config.getboolean('General', 'EnableHealthCheck', fallback=True)
            self.health_check_interval = self.config.getint('General', 'HealthCheckInterval', fallback=60)
            
            # Logging settings
            self.log_level = self.config.get('Logging', 'LogLevel', fallback='INFO')
            self.max_log_files = self.config.getint('Logging', 'MaxLogFiles', fallback=10)
            self.max_log_size = self.config.get('Logging', 'MaxLogSize', fallback='10M')
            
            # Monitoring settings
            self.enable_metrics = self.config.getboolean('Monitoring', 'EnableMetrics', fallback=True)
            self.metrics_port = self.config.getint('Monitoring', 'MetricsPort', fallback=9090)
            self.alert_threshold = self.config.getint('Monitoring', 'AlertThreshold', fallback=3)
            
        except Exception as e:
            print(f"Error loading configuration: {e}")
            sys.exit(1)
    
    def setup_logging(self):
        """Setup logging configuration"""
        log_dir = Path('/var/log/rtc-wakeup')
        log_dir.mkdir(exist_ok=True)
        
        # Configure logging
        log_format = '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
        logging.basicConfig(
            level=getattr(logging, self.log_level.upper()),
            format=log_format,
            handlers=[
                logging.FileHandler(log_dir / 'service.log'),
                logging.StreamHandler(sys.stdout)
            ]
        )
        
        self.logger = logging.getLogger('rtc-wakeup-service')
    
    def signal_handler(self, signum, frame):
        """Handle shutdown signals"""
        self.logger.info(f"Received signal {signum}, shutting down...")
        self.running = False
    
    def validate_rtc_device(self):
        """Validate RTC device accessibility"""
        try:
            if not os.path.exists(self.rtc_device):
                raise FileNotFoundError(f"RTC device not found: {self.rtc_device}")
            
            # Test read access
            with open(self.rtc_device, 'r') as f:
                f.read()
            
            # Test write access
            with open(self.rtc_device, 'w') as f:
                f.write('0')
            
            return True
            
        except Exception as e:
            self.logger.error(f"RTC device validation failed: {e}")
            return False
    
    def calculate_next_wakeup(self):
        """Calculate next wake-up timestamp"""
        try:
            now = datetime.now()
            target_time = datetime.strptime(self.wake_time, '%H:%M:%S').time()
            
            # Create target datetime for today
            target_today = datetime.combine(now.date(), target_time)
            
            # If target time has passed today, use tomorrow
            if target_today <= now:
                target_next = target_today + timedelta(days=1)
            else:
                target_next = target_today
            
            return int(target_next.timestamp())
            
        except Exception as e:
            self.logger.error(f"Error calculating next wake-up: {e}")
            return None
    
    def set_rtc_alarm(self, timestamp):
        """Set RTC alarm with retry logic"""
        for attempt in range(1, self.max_retries + 1):
            try:
                self.logger.info(f"Setting RTC alarm (attempt {attempt}/{self.max_retries})")
                
                # Clear existing alarm
                with open(self.rtc_device, 'w') as f:
                    f.write('0')
                
                # Set new alarm
                with open(self.rtc_device, 'w') as f:
                    f.write(str(timestamp))
                
                # Verify alarm was set
                with open(self.rtc_device, 'r') as f:
                    set_alarm = f.read().strip()
                
                if set_alarm == str(timestamp):
                    wake_time = datetime.fromtimestamp(timestamp)
                    self.logger.info(f"RTC alarm set successfully for: {wake_time}")
                    self.health_status['errors'] = 0
                    return True
                else:
                    self.logger.warning(f"Alarm verification failed: expected {timestamp}, got {set_alarm}")
                    
            except Exception as e:
                self.logger.warning(f"Attempt {attempt} failed: {e}")
            
            if attempt < self.max_retries:
                time.sleep(self.retry_delay)
        
        self.logger.error(f"Failed to set RTC alarm after {self.max_retries} attempts")
        self.health_status['errors'] += 1
        return False
    
    def health_check(self):
        """Perform health check"""
        try:
            # Check RTC device
            if not self.validate_rtc_device():
                self.health_status['status'] = 'unhealthy'
                return False
            
            # Check current alarm
            with open(self.rtc_device, 'r') as f:
                current_alarm = f.read().strip()
            
            if current_alarm and current_alarm != '0':
                alarm_time = datetime.fromtimestamp(int(current_alarm))
                if alarm_time > datetime.now():
                    self.health_status['status'] = 'healthy'
                    self.health_status['last_check'] = datetime.now()
                    return True
            
            self.health_status['status'] = 'no_alarm'
            return False
            
        except Exception as e:
            self.logger.error(f"Health check failed: {e}")
            self.health_status['status'] = 'error'
            return False
    
    def monitor_loop(self):
        """Background monitoring loop"""
        while self.running:
            try:
                if self.enable_health_check:
                    self.health_check()
                    
                    # Check if we need to alert
                    if self.health_status['errors'] >= self.alert_threshold:
                        self.logger.critical(f"Alert: {self.health_status['errors']} consecutive errors")
                
                time.sleep(self.health_check_interval)
                
            except Exception as e:
                self.logger.error(f"Monitor loop error: {e}")
                time.sleep(self.health_check_interval)
    
    def main_loop(self):
        """Main service loop"""
        self.logger.info("Starting main service loop")
        
        while self.running:
            try:
                # Calculate next wake-up time
                next_wakeup = self.calculate_next_wakeup()
                if next_wakeup is None:
                    self.logger.error("Failed to calculate next wake-up time")
                    time.sleep(self.check_interval)
                    continue
                
                # Set RTC alarm
                if self.set_rtc_alarm(next_wakeup):
                    # Wait until it's time to set the next alarm
                    # Check periodically to ensure alarm is still set
                    sleep_time = min(self.check_interval, 3600)  # Max 1 hour
                    time.sleep(sleep_time)
                else:
                    # Failed to set alarm, retry sooner
                    time.sleep(60)
                
            except Exception as e:
                self.logger.error(f"Main loop error: {e}")
                time.sleep(60)
    
    def run(self):
        """Run the service"""
        self.logger.info("Starting RTC Wake-up Service")
        
        # Validate environment
        if not self.validate_rtc_device():
            self.logger.critical("RTC device validation failed, exiting")
            sys.exit(1)
        
        self.running = True
        
        # Start monitoring thread
        if self.enable_health_check:
            monitor_thread = threading.Thread(target=self.monitor_loop, daemon=True)
            monitor_thread.start()
            self.logger.info("Health monitoring started")
        
        # Run main loop
        try:
            self.main_loop()
        except KeyboardInterrupt:
            self.logger.info("Service interrupted by user")
        finally:
            self.running = False
            self.logger.info("RTC Wake-up Service stopped")
 
if __name__ == '__main__':
    service = RTCWakeupService()
    service.run()
EOF

Frequently Asked Questions

Q: Which method should I choose for my iMX6 SabreSD?

A: For development and testing, use Method 1 (Direct Sys Interface) for its simplicity. For production deployments, Method 2 (Script-based) provides the best balance of reliability and maintainability. Choose Method 3 (Service-based) only if you need enterprise-grade monitoring and automatic recovery.

Q: What about power consumption during sleep?

A: The RTC wake-up functionality uses the SNVS (Secure Non-Volatile Storage) domain on iMX6, which operates in the lowest power mode. Current consumption is typically under 10µA, making it suitable for battery-powered applications.

Q: How do I handle leap seconds and time zone changes?

A: The RTC operates in local time, so time zone changes require recalculation of wake-up times. For critical applications, consider using UTC timestamps and converting to local time only for display purposes. Leap seconds are handled automatically by the kernel.

Q: Can I set multiple alarms?

A: The standard RTC interface supports only one alarm at a time. For multiple wake-up times, you’ll need to implement a scheduling system that sets the next alarm after each wake-up event, or use additional timer sources like GPIO-based external RTCs.

Q: What happens if the system doesn’t wake up?

A: Common causes include incorrect power management configuration, disabled wake-up sources, or hardware issues. Verify that the RTC is configured as a wake-up source in /sys/class/rtc/rtc0/device/power/wakeup and check system logs for power management events.

Q: How accurate is the RTC wake-up timing?

A: The iMX6 SNVS RTC typically provides accuracy within ±20ppm (about 1.7 seconds per day) at room temperature. For higher precision, consider external temperature-compensated RTCs or regular NTP synchronization when the system is awake.
