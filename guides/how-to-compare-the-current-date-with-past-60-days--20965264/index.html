How to Compare the Current Date with Past 60 Days Using Shell Script: 2 Methods + Complete Guide

Quick Answer

Use epoch time conversion for reliable date comparison across all Unix systems:

# Get current time and 60 days ago in seconds
now=$(date +%s)
sixty_days_ago=$((now - 60*24*3600))
 
# Compare your timestamp
start_time_sec=$(date -d "$start_time" +%s)
if [ $start_time_sec -ge $sixty_days_ago ]; then
    echo "Date is within last 60 days"
fi

Complete Log Processing Script

#!/bin/bash
 
# Function to check if date is within last 60 days
is_within_60_days() {
    local input_date="$1"
    local now=$(date +%s)
    local sixty_days_ago=$((now - 60*24*3600))
    local input_timestamp=$(date -d "$input_date" +%s 2>/dev/null)
    
    if [ $? -ne 0 ]; then
        echo "Invalid date format: $input_date" >&2
        return 1
    fi
    
    [ $input_timestamp -ge $sixty_days_ago ]
}
 
# Process log file line by line
while IFS= read -r line; do
    # Extract timestamp from log line (YYYY-MM-DD HH:MM:SS format)
    start_time=$(echo "$line" | sed -e 's/^\([0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9] [0-9][0-9]:[0-9][0-9]:[0-9][0-9]\).*/\1/')
    
    if is_within_60_days "$start_time"; then
        echo "Recent: $line"
    else
        echo "Old: $line"
    fi
done < logfile.txt

Method 1: Epoch Time Comparison (Recommended)

Rating: ⭐⭐⭐⭐⭐ Recommended

How it works:

This method converts dates to Unix timestamps (seconds since January 1, 1970) for mathematical comparison. It’s the most reliable approach because it handles time zones, leap years, and different month lengths automatically.

Complete Implementation:

#!/bin/bash
 
# Enhanced date comparison with error handling and logging
compare_date_with_60_days() {
    local input_line="$1"
    local debug_mode="${2:-false}"
    
    # Extract timestamp using improved regex
    local start_time=$(echo "$input_line" | sed -n 's/.*\([0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\} [0-9]\{2\}:[0-9]\{2\}:[0-9]\{2\}\).*/\1/p')
    
    # Validate extracted timestamp
    if [ -z "$start_time" ]; then
        echo "ERROR: No valid timestamp found in line: $input_line" >&2
        return 2
    fi
    
    # Convert to epoch time with error checking
    local start_time_sec=$(date -d "$start_time" +%s 2>/dev/null)
    if [ $? -ne 0 ]; then
        echo "ERROR: Invalid date format: $start_time" >&2
        return 3
    fi
    
    # Calculate current time and 60-day threshold
    local now=$(date +%s)
    local sixty_days_in_seconds=$((60 * 24 * 3600))  # 60 days * 24 hours * 3600 seconds
    local threshold_time=$((now - sixty_days_in_seconds))
    
    # Debug output if requested
    if [ "$debug_mode" = "true" ]; then
        echo "DEBUG: Extracted time: $start_time" >&2
        echo "DEBUG: Start timestamp: $start_time_sec" >&2
        echo "DEBUG: Current timestamp: $now" >&2
        echo "DEBUG: Threshold timestamp: $threshold_time" >&2
        echo "DEBUG: Time difference: $((now - start_time_sec)) seconds" >&2
    fi
    
    # Perform comparison
    if [ $start_time_sec -ge $threshold_time ]; then
        echo "RECENT: $input_line"
        return 0
    else
        echo "OLD: $input_line"
        return 1
    fi
}
 
# Example usage with different input formats
test_lines=(
    "2024-01-15 10:30:45 Application started successfully"
    "2023-12-01 08:15:22 User login detected"
    "2024-02-20 14:22:33 Error: Connection timeout"
)
 
echo "=== Testing Date Comparison ==="
for line in "${test_lines[@]}"; do
    compare_date_with_60_days "$line" true
    echo "---"
done
 
# Batch processing function for large files
process_log_file() {
    local filename="$1"
    local output_recent="${2:-recent_entries.log}"
    local output_old="${3:-old_entries.log}"
    
    if [ ! -f "$filename" ]; then
        echo "ERROR: File $filename not found" >&2
        return 1
    fi
    
    local recent_count=0
    local old_count=0
    
    # Clear output files
    > "$output_recent"
    > "$output_old"
    
    echo "Processing $filename..."
    
    while IFS= read -r line || [ -n "$line" ]; do
        if compare_date_with_60_days "$line" false >/dev/null; then
            echo "$line" >> "$output_recent"
            ((recent_count++))
        else
            echo "$line" >> "$output_old"
            ((old_count++))
        fi
    done < "$filename"
    
    echo "Processing complete:"
    echo "  Recent entries (last 60 days): $recent_count"
    echo "  Older entries: $old_count"
    echo "  Output files: $output_recent, $output_old"
}

Method 2: ISO Date String Comparison

Rating: ⭐⭐⭐⭐ Good Alternative

How it works:

This method leverages the fact that ISO 8601 date format (YYYY-MM-DD) can be compared lexicographically as strings. It’s simpler but has limitations with time components.

Complete Implementation:

#!/bin/bash
 
# ISO date string comparison method
compare_iso_dates() {
    local input_line="$1"
    local include_time="${2:-false}"
    
    if [ "$include_time" = "true" ]; then
        # Extract full datetime (YYYY-MM-DD HH:MM:SS)
        local start_datetime=$(echo "$input_line" | grep -o '[0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\} [0-9]\{2\}:[0-9]\{2\}:[0-9]\{2\}')
        local start_date=$(echo "$start_datetime" | cut -d' ' -f1)
    else
        # Extract just date (YYYY-MM-DD)
        local start_date=$(echo "$input_line" | grep -o '[0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\}')
    fi
    
    if [ -z "$start_date" ]; then
        echo "ERROR: No valid date found in: $input_line" >&2
        return 1
    fi
    
    # Get current date and 60 days ago in ISO format
    local current_date=$(date +%Y-%m-%d)
    local sixty_days_ago=$(date +%Y-%m-%d -d '60 days ago')
    
    echo "Comparing: $start_date with threshold: $sixty_days_ago"
    
    # String comparison (works because ISO format is lexicographically ordered)
    if [[ "$start_date" > "$sixty_days_ago" ]] || [[ "$start_date" = "$sixty_days_ago" ]]; then
        echo "RECENT: $input_line"
        return 0
    else
        echo "OLD: $input_line"
        return 1
    fi
}
 
# Advanced ISO comparison with time precision
compare_iso_with_time() {
    local input_line="$1"
    
    # Extract full timestamp
    local timestamp=$(echo "$input_line" | grep -o '[0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\} [0-9]\{2\}:[0-9]\{2\}:[0-9]\{2\}')
    
    if [ -z "$timestamp" ]; then
        echo "ERROR: No valid timestamp found" >&2
        return 1
    fi
    
    # Get current datetime and 60 days ago
    local current_datetime=$(date '+%Y-%m-%d %H:%M:%S')
    local sixty_days_ago_datetime=$(date '+%Y-%m-%d %H:%M:%S' -d '60 days ago')
    
    # For precise comparison, we still need epoch conversion
    local timestamp_epoch=$(date -d "$timestamp" +%s)
    local threshold_epoch=$(date -d "$sixty_days_ago_datetime" +%s)
    
    if [ $timestamp_epoch -ge $threshold_epoch ]; then
        echo "RECENT: $input_line"
        return 0
    else
        echo "OLD: $input_line"
        return 1
    fi
}
 
# Utility function for date validation
validate_iso_date() {
    local date_string="$1"
    
    # Check format with regex
    if [[ ! "$date_string" =~ ^[0-9]{4}-[0-9]{2}-[0-9]{2}$ ]]; then
        return 1
    fi
    
    # Validate actual date
    date -d "$date_string" >/dev/null 2>&1
    return $?
}
 
# Example usage
echo "=== ISO Date Comparison Examples ==="
 
test_lines=(
    "Log entry from 2024-01-15 10:30:45"
    "System event on 2023-11-20 14:22:33"
    "2024-02-28 09:15:00 - Database backup completed"
)
 
for line in "${test_lines[@]}"; do
    echo "Testing: $line"
    compare_iso_dates "$line" true
    echo "---"
done

Common Problems & Solutions

Problem 1: Invalid Date Format Error

Symptoms: date: invalid date error when parsing timestamps

Solution:

# Robust date parsing with fallback
parse_date_safely() {
    local date_string="$1"
    local formats=(
        "%Y-%m-%d %H:%M:%S"
        "%Y/%m/%d %H:%M:%S"
        "%m/%d/%Y %H:%M:%S"
        "%d-%m-%Y %H:%M:%S"
    )
    
    for format in "${formats[@]}"; do
        if parsed_date=$(date -d "$date_string" +%s 2>/dev/null); then
            echo "$parsed_date"
            return 0
        fi
    done
    
    echo "ERROR: Unable to parse date: $date_string" >&2
    return 1
}

Problem 3: Large File Processing Performance

Symptoms: Slow processing of large log files

Solution:

# Optimized batch processing
process_large_file() {
    local filename="$1"
    local batch_size=1000
    local line_count=0
    local batch_lines=()
    
    while IFS= read -r line; do
        batch_lines+=("$line")
        ((line_count++))
        
        if [ $((line_count % batch_size)) -eq 0 ]; then
            # Process batch
            for batch_line in "${batch_lines[@]}"; do
                compare_date_with_60_days "$batch_line"
            done
            batch_lines=()
            echo "Processed $line_count lines..." >&2
        fi
    done < "$filename"
    
    # Process remaining lines
    for batch_line in "${batch_lines[@]}"; do
        compare_date_with_60_days "$batch_line"
    done
}

stream_process() {

Log Rotation and Cleanup

#!/bin/bash
# Enterprise log management script
 
LOG_RETENTION_DAYS=60
LOG_DIRECTORY="/var/log/application"
ARCHIVE_DIRECTORY="/var/log/archive"
 
cleanup_old_logs() {
    local retention_days="$1"
    local log_dir="$2"
    local archive_dir="$3"
    
    # Create archive directory if it doesn't exist
    mkdir -p "$archive_dir"
    
    # Find and process log files
    find "$log_dir" -name "*.log" -type f | while read -r logfile; do
        # Get file modification time
        file_date=$(stat -c %Y "$logfile")
        current_time=$(date +%s)
        age_days=$(( (current_time - file_date) / 86400 ))
        
        if [ $age_days -gt $retention_days ]; then
            echo "Archiving old log: $logfile (age: $age_days days)"
            gzip "$logfile"
            mv "$logfile.gz" "$archive_dir/"
        fi
    done
}
 
# Usage
cleanup_old_logs $LOG_RETENTION_DAYS "$LOG_DIRECTORY" "$ARCHIVE_DIRECTORY"

Database Backup Validation

#!/bin/bash
# Validate database backups within retention period
 
validate_backup_freshness() {
    local backup_dir="$1"
    local max_age_days="${2:-1}"  # Default 1 day for critical systems
    
    latest_backup=$(find "$backup_dir" -name "*.sql.gz" -type f -printf '%T@ %p\n' | sort -n | tail -1)
    
    if [ -z "$latest_backup" ]; then
        echo "CRITICAL: No backups found in $backup_dir"
        return 1
    fi
    
    backup_timestamp=$(echo "$latest_backup" | cut -d' ' -f1 | cut -d'.' -f1)
    current_timestamp=$(date +%s)
    age_seconds=$((current_timestamp - backup_timestamp))
    age_days=$((age_seconds / 86400))
    
    if [ $age_days -le $max_age_days ]; then
        echo "OK: Latest backup is $age_days days old"
        return 0
    else
        echo "WARNING: Latest backup is $age_days days old (max: $max_age_days)"
        return 1
    fi
}

Session Cleanup Script

#!/bin/bash
# Clean up expired web sessions
 
SESSION_DIR="/tmp/php_sessions"
SESSION_LIFETIME_HOURS=24
 
cleanup_expired_sessions() {
    local session_dir="$1"
    local lifetime_hours="$2"
    local lifetime_seconds=$((lifetime_hours * 3600))
    local current_time=$(date +%s)
    local threshold_time=$((current_time - lifetime_seconds))
    
    local cleaned_count=0
    local total_count=0
    
    for session_file in "$session_dir"/sess_*; do
        if [ -f "$session_file" ]; then
            ((total_count++))
            file_mtime=$(stat -c %Y "$session_file")
            
            if [ $file_mtime -lt $threshold_time ]; then
                rm "$session_file"
                ((cleaned_count++))
            fi
        fi
    done
    
    echo "Session cleanup complete: $cleaned_count/$total_count files removed"
}

Cache Invalidation

#!/bin/bash
# Invalidate cache files older than specified time
 
invalidate_cache() {
    local cache_dir="$1"
    local max_age_minutes="${2:-60}"
    local max_age_seconds=$((max_age_minutes * 60))
    local current_time=$(date +%s)
    
    find "$cache_dir" -name "*.cache" -type f | while read -r cache_file; do
        file_mtime=$(stat -c %Y "$cache_file")

        if [ $age_seconds -gt $max_age_seconds ]; then
            echo "Invalidating cache: $cache_file"
            rm "$cache_file"
        fi
    done
}

Performance-Critical Situations

High-Frequency Log Processing

#!/bin/bash
# Optimized for processing millions of log entries
 
# Pre-calculate threshold once
SIXTY_DAYS_AGO=$(date -d '60 days ago' +%s)
 
process_high_volume_logs() {
    local log_file="$1"
    local output_recent="$2"
    local output_old="$3"
    
    # Use awk for faster processing
    awk -v threshold="$SIXTY_DAYS_AGO" '
    {
        # Extract timestamp (assuming first field is ISO datetime)
        cmd = "date -d \"" $1 " " $2 "\" +%s 2>/dev/null"
        cmd | getline timestamp
        close(cmd)
        
        if (timestamp >= threshold) {
            print > "'"$output_recent"'"
        } else {
            print > "'"$output_old"'"
        }
    }' "$log_file"
}

# Check if file is newer than X days
is_file_recent() {
    local filename="$1"
    local max_age_days="$2"
    
    if [ ! -f "$filename" ]; then
        return 1
    fi
    
    local file_age=$(stat -c %Y "$filename")

}

Summary

  1. Use Epoch Time Method for most production scenarios - it’s accurate, reliable, and handles edge cases
  2. Consider ISO String Method for simple date-only comparisons and human-readable output
  3. Test thoroughly in your specific environment, especially with different date formats
  4. Follow best practices including error handling, input validation, and performance optimization
  5. Consider performance implications when processing large datasets - use streaming approaches

  • Always validate input dates before processing
  • Handle timezone considerations for distributed systems
  • Optimize for your specific use case (accuracy vs. performance)
  • Include proper error handling and logging

Frequently Asked Questions

Q: Which method should I choose?

A: For most applications, use the Epoch Time method (Method 1). It’s more accurate, handles edge cases better, and works consistently across different systems. Use the ISO String method only for simple date-only comparisons where human readability is more important than precision.

Q: What about shell compatibility?

A: The Epoch Time method works on all POSIX-compliant shells with the date command. However, the -d option might not be available on some minimal systems (like Alpine Linux). In such cases, you may need to use alternative approaches or install GNU coreutils.

Q: How do I handle edge cases?

A: Always include input validation, error handling, and consider timezone issues. Use the robust parsing functions provided in the examples above. Test with various date formats and edge cases like leap years and month boundaries.

Q: Are there performance differences?

A: Yes, the ISO String method is slightly faster for simple comparisons, but the Epoch Time method is more efficient for complex date arithmetic. For high-volume processing, consider using awk or other optimized tools as shown in the performance-critical examples.

Q: Can I combine multiple approaches?
